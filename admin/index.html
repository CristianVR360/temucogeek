Script SQL para Supabase

Ejecuta este código en el SQL Editor de tu consola Supabase para crear la tabla de postulaciones: 

CREATE TABLE IF NOT EXISTS postulaciones (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL,
  nombre_expositor TEXT NOT NULL,
  nombre_marca TEXT NOT NULL,
  redes_sociales TEXT NOT NULL,
  telefono TEXT NOT NULL,
  email TEXT NOT NULL,
  ciudad TEXT NOT NULL,
  categorias TEXT NOT NULL,
  espacio_tipo TEXT NOT NULL,
  is_formalizado BOOLEAN DEFAULT false,
  rut_empresa TEXT,
  razon_social TEXT,
  giro_empresa TEXT,
  descripcion_productos TEXT NOT NULL,
  estado TEXT DEFAULT 'pendiente',
  notas_internas TEXT
);

ALTER TABLE postulaciones ENABLE ROW LEVEL SECURITY;

📧 Configuración de Notificaciones por Correo vía Supabase Webhook (Sin CORS)

Los navegadores bloquean llamadas AJAX directas a la API de Resend por seguridad (CORS). Para que Supabase envíe el correo automáticamente en el servidor al registrarse una postulación:

  1. Ve a tu consola de Supabase → Database → Webhooks.
  2. Haz clic en Create a new webhook.
  3. Name: notificar_resend
  4. Table: postulaciones (Event: INSERT)
  5. Webhook Type: HTTP Request
  6. Method: POST → URL: https://api.resend.com/emails
  7. HTTP Headers:
    Authorization: Bearer TU_NUEVA_RESEND_API_KEY
    Content-Type: application/json
